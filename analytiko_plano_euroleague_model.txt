Παρακάτω είναι ένα **ακριβές, βήμα‑βήμα πλάνο** για prediction + simulations για την **φετινή EuroLeague (2025–26)**, με **Euroleague API v3** και το wrapper repo του Georgios Giasemidis. Στόχος: να πατάς “run” και να βγάζεις πιθανότητες/προβλέψεις για **την επόμενη αγωνιστική (Round)**.

---

## 0) Κλείδωσε identifiers (για να μη χαθείς)
- **competitionCode**: `E` (EuroLeague)
- **seasonCode**: `E2025` (start year 2025)
- **season (int)** για wrapper: `2025`

Το API v3 έχει base: `https://api-live.euroleague.net/v3` και endpoints για clubs, standings, και για κάθε game `report / stats / teamsComparison`.

## 2) Data model (τι θα αποθηκεύεις – με primary keys)
Χρησιμοποίησε **DuckDB** (1 αρχείο `data/el.db`) ή Parquet.

### 2.1 Πίνακες
1) **clubs**
- `teamCode` (string)
- `clubCode` (string)
- `name` (string)

2) **games** (1 row ανά game)
- `season` (int, π.χ. 2025)
- `gameCode` (string, π.χ. `E2025_47`)
- `gameNumber` (int, π.χ. 47)
- `roundNumber` (int)
- `dateTime` (datetime)
- `homeTeamCode`, `awayTeamCode`
- `status` (Played/NotPlayed)
- `homePts`, `awayPts`

3) **team_game_stats** (2 rows ανά game: μία ανά ομάδα)
- PK: (`season`, `gameNumber`, `teamCode`)
- fields που χρειάζεσαι για possessions:
  - `PTS, FGA, FTA, OREB, TO` (+ ό,τι άλλο θες)

4) **team_ratings_snapshot** (ανά round ή “as-of date”)
- PK: (`season`, `asOfRound`, `teamCode`)
- `OffRtg, DefRtg, NetRtg`
- `HomeNetRtg, AwayNetRtg`
- `EloCurrent, EloHistBlend`

5) **predictions_round**
- PK: (`season`, `roundNumber`, `gameNumber`)
- `pHomeWin`, `muMargin`, `sigmaMargin`
- quantiles: `q10,q50,q90`

## 3) Ingestion – πώς τραβάς τα δεδομένα (μεθοδικά)
### 3.1 Clubs / Teams (v3)
- Κάνε call στο v3 endpoint `/clubs` και κράτα mapping κωδικών.

**Κανόνας:** Από εδώ και πέρα δουλεύεις εσωτερικά με `teamCode` (σταθερό join key).

---

### 3.2 Games list + Round info (για schedule/next round)
Έχεις 2 ασφαλείς επιλογές:

**Option A (καθαρά v3, recommended):**
- Χρησιμοποίησε v3 endpoint για games/schedules (στο swagger) και πάρε:
  - `roundNumber`, `gameCode`, `home/away`, ημερομηνία.

**Option B (γρήγορο workaround για game list):**
- Πάρε games από `/v1/results?seasonCode=E2025` (είναι XML αλλά δίνει σίγουρα `gamecode`, `gameday`, date).
- Μετά, για τα αναλυτικά δεδομένα (report/stats) δουλεύεις με v3/wrapper.

> Πρακτικά: κρατάς `gameCode="E2025_47"` και βγάζεις `gameNumber=47` (split στο `_`).

---

### 3.3 Game reports / stats (v3)
Για να χτίσεις **results + features**, χρειάζεσαι *τουλάχιστον*:
- **Game report** (τελικό σκορ, status, κλπ)
- **Game stats** (team totals για possessions)
- (optional) **teamsComparison** (pregame context)

Αυτά υπάρχουν σε v3:
- `/competitions/{competitionCode}/seasons/{seasonCode}/games/{gameCode}/report`
- `/.../stats`
- `/.../teamsComparison`

**Με το wrapper του Giasemidis (σου γλιτώνει endpoints):**
- `GameStats.get_game_report(season, game_code)`
- `GameStats.get_game_stats(season, game_code)`
- `GameStats.get_game_teams_comparison(season, game_code)`

### 3.4 Metadata (Round, venue κλπ) – χρήσιμο για “next round”
Το wrapper έχει `GameMetadata` που καλεί `https://live.euroleague.net/api/Header` με params `gamecode` και `seasoncode=f"{competition}{season}"` και επιστρέφει και `Round`.

**Χρήση:**
- `GameMetadata.get_game_metadata_single_season(2025)` → παίρνεις `Round` για όλα τα games.

## 4) Μετατροπές / Curated layer (κανόνες καθαρισμού)
Για κάθε game:
1) **Εντόπισε αν παίχτηκε**
- Από report: `homePts/awayPts` όχι null και status “Finished/Played”.
2) **Team stats normalization**
- Ενοποίησε ονόματα στηλών (π.χ. `FGA`, `FTA`, `OREB`, `TO`, `PTS`).
3) **Δημιούργησε 2 rows per game** (home + away) στο `team_game_stats`.
4) **Δημιούργησε opponent join**
- για κάθε (`season`, `gameNumber`) κάνε self-join ώστε η home γραμμή να “βλέπει” τα away stats (για DefRtg).

---

## 5) Υπολογισμός Net Rating (this year performance) + home/away split
### 5.1 Possessions (ανά team-game)
Χρησιμοποίησε estimate:
\[
poss = FGA + 0.44 \cdot FTA - OREB + TO
\]

### 5.2 Ratings (ανά 100 possessions)
- `OffRtg = 100 * PTS / poss`
- `DefRtg = 100 * OppPTS / OppPoss`
- `NetRtg = OffRtg - DefRtg`

### 5.3 Home/Away split
Φτιάξε 3 aggregates “as of round R”:
- overall: όλα τα played games ≤ R
- home: played games όπου team ήταν home
- away: played games όπου team ήταν away

**Tip σταθερότητας:** early season βάλε shrinkage:
\[
NetRtg_{adj} = \frac{n}{n+k} NetRtg + \frac{k}{n+k} 0
\]
με `k≈5–8` games.

---

## 6) Historic weight (Elo από past 2 seasons) + merge στη φετινή
### 6.1 Δεδομένα Elo training set
Πάρε αποτελέσματα από:
- σεζόν start **2023** (2023–24)
- σεζόν start **2024** (2024–25)
με ίδιο ingestion (reports + winner).

### 6.2 Elo update rules (συγκεκριμένα defaults)
- αρχικό: `Elo=1500` για όλους
- home advantage: `HCA = +65 Elo points` (σταθερό)
- K-factor: `K=20` (ή 25 για αρχές season αν θες)

Expected prob:
\[
p = \frac{1}{1 + 10^{-((Elo_H + HCA) - Elo_A)/400}}
\]
Update:
- `Elo_H += K*(S - p)`
- `Elo_A -= K*(S - p)` όπου `S=1` αν home win else 0

### 6.3 Blend “past 2 seasons” (αυτό που ζήτησες)
Στο τέλος των 2 σεζόν:
- `EloHist = 0.65*Elo_2024 + 0.35*Elo_2023`

### 6.4 Φετινό EloCurrent
- αρχικοποίησε 2025–26: `EloCurrent(team)=EloHist(team)`
- μετά κάνε update game-by-game με τα φετινά αποτελέσματα.

## 7) Projection model για επόμενη αγωνιστική (Round)
### 7.1 Εντοπισμός “επόμενης αγωνιστικής”
Από `games` table:
- Ορισμός: **NextRound = μικρότερο round όπου υπάρχει έστω 1 game NotPlayed**
- Εναλλακτικά: `maxRoundFullyPlayed + 1`

### 7.2 Features για κάθε matchup (home vs away)
Για round = NextRound, φτιάξε:
- `A = NetRtgHomeSplit(homeTeam) - NetRtgAwaySplit(awayTeam)`
- `B = (EloCurrent(home)-EloCurrent(away))/25`  (κλίμακα)
- (προαιρετικό) `RestDiff`, `Travel` κλπ (άστο για v2)

### 7.3 Probability model (logistic)
\[
score = w_1 A + w_2 B + w_3
\]
όπου `w3` είναι σταθερός όρος home edge (ή βάλε το μέσα στο Elo με HCA).

\[
P(HomeWin) = \sigma(score)
\]

**Αρχικές τιμές (χωρίς fitting):**
- `w1=0.08` (NetRtg είναι “ανά 100 poss”, άρα θέλει μικρό βάρος)
- `w2=0.9`
- `w3=0.12`

### 7.4 Calibration (για να μην είναι “overconfident”)
Κάνε backtest στις δύο προηγούμενες σεζόν:
- walk-forward ανά round
- metric: logloss + Brier
- ρύθμισε `w1,w2,w3`

---

## 8) Monte Carlo simulations (για score/margin distributions)
### 8.1 Μετατροπή score → expected margin
Ορίζεις:
\[
\mu = \alpha_1 A + \alpha_2 (EloDiff/25) + \alpha_3
\]
και παίρνεις σ από ιστορικά margins (std των τελικών διαφορών last 2 seasons).

**Defaults:**
- `alpha1=0.9`
- `alpha2=1.0`
- `alpha3=1.5` (πόντοι home edge)
- `sigma=11.5` (ξεκίνα έτσι και μετά το μετράς από data)

### 8.2 Simulation ανά παιχνίδι
Για `N=20,000`:
- `margin ~ Normal(mu, sigma)`
- home win αν `margin>0`
- κράτα:
  - `pHomeWin = mean(margin>0)`
  - `meanMargin`
  - quantiles (10/50/90)

Αποθήκευσε σε `predictions_round`.

---

## 9) “Run” pipeline (ακριβής ροή εκτέλεσης)
Κάθε φορά που θες predictions:
1) `fetch_clubs` (refresh 1 φορά/βδομάδα)
2) `fetch_games` (ή from v3 swagger endpoint ή από v1 results)
3) `fetch_game_reports` για όλα τα games (ή increment μόνο τα νέα)
4) `fetch_game_stats` μόνο για played games
5) `build_team_game_stats` + possessions
6) `compute_net_ratings` + splits (as-of latest round)
7) `update_elo` (hist computed once, current updated incremental)
8) `compute_next_round_predictions`
9) `run_monte_carlo`
10) export:
   - CSV: `outputs/round_{R}_predictions.csv`
   - optional simple HTML report

---

## 10) Πρακτικά “gotchas” (να μη σπάει)
- **Game codes:** Αν έχεις `E2025_47`, ο wrapper πολλές φορές θέλει `game_code=47` (int) και `season=2025`. Από v1 results βλέπεις και τα δύο (`gamecode`, `gamenumber`).
- **Round 1 teamsComparison:** στο wrapper αναφέρεται ότι το `teamsComparison` είναι pre-game και μπορεί να μην υπάρχει για round 1. Μην το κάνεις hard dependency.
- **Caching:** κράτα raw JSON responses σε `data/raw/` με filename `season_round_endpoint_timestamp.json` για reproducibility.
- **Consistency:** πάντα join με `teamCode`, όχι με εμφανιζόμενα ονόματα ομάδων.
